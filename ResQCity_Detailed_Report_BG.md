# Доклад за платформата ResQCity

## 1. Въведение и цел на системата

ResQCity е интегрирана платформа за управление на градски инциденти, сигнали и ресурси, предназначена да подпомага граждани, администрация и аварийни екипи. Основната цел е да се осигури бърза, прозрачна и ефективна комуникация между всички заинтересовани страни при възникване на инциденти, аварии или нужда от помощ в градска среда.

Платформата предоставя:
- Възможност за подаване и проследяване на сигнали от граждани
- Управление на ресурси и екипи в реално време
- Интерактивна карта с множество слоеве (инциденти, превозни средства, магазини, метео)
- Административен панел за анализ и контрол
- Интеграция с външни системи (метео, имейл, КЕП, Mapbox)

## 2. Архитектура и използвани технологии

### 2.1. Архитектурен модел

ResQCity е проектирана като съвременна уеб платформа с ясно разделение между клиентска част (frontend), сървърна част (backend) и база данни. Този подход позволява лесна поддръжка, мащабируемост и интеграция с външни услуги. Използва се монорепо структура, което улеснява управлението на зависимостите и версиите на всички компоненти.

**Избор на технологии и обосновка:**

- **Next.js 14** – Next.js е водещ фреймуърк за React приложения, който комбинира сървърно рендиране (SSR) и статично генериране (SSG). Това осигурява бързо зареждане, SEO оптимизация и добра производителност. Версия 14 предоставя най-новите възможности и оптимизации.

- **React 18** – React е най-популярната библиотека за изграждане на потребителски интерфейси. Версия 18 въвежда подобрена конкурентност и оптимизации за по-добро потребителско изживяване.

- **TypeScript** – Използва се за статична типизация, което намалява грешките при разработка, улеснява поддръжката и подобрява четимостта на кода.

- **Tailwind CSS** – Tailwind позволява бързо и последователно стилизиране на интерфейса чрез utility-first подход. Това ускорява разработката и гарантира модерен, адаптивен дизайн.

- **Next.js API Routes & Node.js** – API Routes в Next.js позволяват изграждане на backend логика в същия проект, което улеснява интеграцията между frontend и backend. Node.js е стабилна и широко използвана среда за сървърни приложения.

- **Socket.IO (WebSocket)** – Избран заради лесната интеграция с Node.js и възможността за двупосочна комуникация в реално време (напр. проследяване на превозни средства, live обновяване на карта).

- **PostgreSQL (Supabase)** – PostgreSQL е мощна релационна база данни, подходяща за сложни заявки и големи обеми данни. Supabase предоставя cloud-hosted PostgreSQL с лесна интеграция и управление.

- **Prisma ORM** – Prisma осигурява типобезопасен достъп до базата данни, автоматични миграции и лесно seed-ване на тестови данни. Това ускорява разработката и намалява риска от грешки при работа с данни.

- **JWT (JSON Web Token)** – JWT е стандарт за сигурна автентикация и управление на сесии. Позволява лесно разграничаване на роли и защита на API.

- **bcryptjs** – Използва се за сигурно хеширане на пароли, което е критично за защитата на потребителските данни.

- **Mapbox (по избор), custom grid overlay** – Mapbox е водеща платформа за интерактивни карти, но системата поддържа и custom grid overlay за по-голяма гъвкавост и независимост от външни услуги.

- **OpenWeatherMap API** – Избран заради лесната интеграция и богатите метео данни, които се визуализират на картата и в sidebar.

- **Nodemailer, SMTP fallback** – Nodemailer е стандарт за изпращане на имейли в Node.js. SMTP fallback гарантира, че съобщенията ще бъдат доставени дори при проблем с основния доставчик.

### 2.2. Основни компоненти

- **API слой** – RESTful endpoints за автентикация, сигнали, магазини, публични данни. Позволява лесно разширяване и интеграция с външни системи.
- **WebSocket слой** – Реално време за проследяване на превозни средства и обновяване на карта. Използва Socket.IO за надеждна комуникация.
- **UI компоненти** – Модулни React компоненти за форми, карти, таблици, sidebar-и. Това улеснява повторната употреба и поддръжката на интерфейса.
- **DevOps** – Docker и docker-compose за лесно стартиране и деплоймънт, Vercel/Cloud deployment за автоматичен build и CI/CD pipelines за качество и бързина на доставката.

## 3. Основни функционалности

### 3.1. Граждански модул

- Регистрация и вход с роля CITIZEN
- Подаване на сигнал (инцидент) с координати, описание, снимки
- Проследяване на статус на сигналите
- Достъп до публична карта с инциденти и ресурси

### 3.2. Административен модул

- Вход с роля ADMIN/SUPER_ADMIN
- Преглед и обработка на всички сигнали
- Управление на екипи, превозни средства, магазини
- Статистики, heatmap, анализи
- Управление на категории, институции, отговорни лица

### 3.3. Интерактивна карта

- Многослойна визуализация: инциденти, превозни средства, магазини, метео
- Реално време: движение на превозни средства, обновяване на статуси
- Филтри и търсене по категории, статус, локация
- Възможност за добавяне на нови слоеве (BG-Alert, социални услуги и др.)

### 3.4. Магазини и инвентар

- Публичен API за наличности в магазини
- Визуализация на карта с реални/тестови данни
- Възможност за интеграция с външни доставчици

### 3.5. Метео интеграция

- Вземане на данни от OpenWeatherMap API
- Визуализация на текущи условия и прогноза на картата

## 4. База данни и модели

### 4.1. Основни модели (Prisma schema)

- **User**: id, email, password, role, createdAt, updatedAt
- **Report**: id, title, description, status, location, images, userId, createdAt, updatedAt
- **Comment**: id, text, reportId, userId, createdAt
- **ReportHistory**: id, reportId, status, changedBy, changedAt
- **Store**: id, name, location, inventory
- **Vehicle**: id, type, status, location, lastUpdate

### 4.2. Връзки и релации

- Един потребител може да има много сигнали
- Всеки сигнал може да има история и коментари
- Магазините и превозните средства са отделни ентитети, свързани с карта

## 12. Подробен архитектурен преглед

### 12.1. Логическа архитектура

Платформата е разделена на три основни слоя:
- **Клиентски слой (Frontend)**: Next.js SPA, SSR, React компоненти, Tailwind CSS
- **Сървърен слой (Backend/API)**: Next.js API Routes, Node.js, Socket.IO, бизнес логика
- **База данни**: PostgreSQL (Supabase), управлявана чрез Prisma ORM

#### Диаграма (описателно):

```
Потребител <-> Next.js (React) <-> API Routes (Node.js) <-> Prisma ORM <-> PostgreSQL (Supabase)
																 ^
																 |
													 Socket.IO (WebSocket)
```

### 12.2. Поток на данни

1. Потребителят подава сигнал през уеб интерфейса
2. Данните се валидират на клиента и се изпращат към API
3. API валидира, съхранява в базата и връща потвърждение
4. При нужда се изпраща известие по WebSocket към администраторите
5. Администратор обработва сигнала, статусът се обновява в реално време

### 12.3. Слоев модел

- **Презентационен слой**: UI, локализация, достъпност
- **Бизнес логика**: Валидации, ролеви проверки, обработка на сигнали
- **Данни**: ORM, миграции, seed, бекъп

## 13. Примерни потребителски сценарии (User Stories)

### 13.1. Гражданин подава сигнал
1. Влиза в системата с профил "Гражданин"
2. Попълва форма за инцидент: локация, описание, снимка
3. Получава потвърждение и може да следи статуса

### 13.2. Администратор обработва сигнал
1. Влиза с профил "Администратор"
2. Вижда списък с нови сигнали
3. Преглежда детайли, добавя коментар, сменя статус
4. Сигналът се обновява в реално време за гражданина

### 13.3. Диспечер следи превозни средства
1. Влиза в модул "Диспечер"
2. Следи карта с реално време движение на екипи
3. Изпраща инструкции към екипите

## 14. API документация (примерни заявки и отговори)

### 14.1. Регистрация на потребител

**POST /api/auth/register**
```
{
	"email": "user@example.com",
	"password": "password123"
}
```
**Отговор:**
```
{
	"token": "<jwt>",
	"user": { "id": 1, "email": "user@example.com", "role": "CITIZEN" }
}
```

### 14.2. Подаване на сигнал

**POST /api/reports**
```
{
	"title": "Паднало дърво",
	"description": "На ул. Иван Вазов има паднало дърво.",
	"location": { "lat": 42.6977, "lng": 23.3219 },
	"images": ["/uploads/incidents/tree.jpg"]
}
```
**Отговор:**
```
{
	"id": 123,
	"status": "NEW",
	"createdAt": "2026-03-26T10:00:00Z"
}
```

### 14.3. Получаване на публични сигнали

**GET /api/reports/public**
**Отговор:**
```
[
	{
		"id": 123,
		"title": "Паднало дърво",
		"status": "IN_PROGRESS",
		"location": { "lat": 42.6977, "lng": 23.3219 }
	},
	...
]
```
